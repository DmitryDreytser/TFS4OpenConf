
Данный скрипт предназначен для приведения программных модулей к общему знаменателю. 
В соответствии со структурой программы расставляются отступы, пустые строки, пробелы.
Настройка параметров делается прямо в тексте скрипта, там же и описание.

Скрипт опробован на NS Perl (http://people.netscape.com/richm/nsPerl, ~3Mb, без
документации) и ActivePerl (http://www.activestate.com/ActivePerl, ~12Mb, с очень
хорошей документацией). На cygwin Perl скрипт почему-то не включается локализация, и 
скрипт будет работать приаильно только в том случае, если все ключевые слова написаны
так же, как в regex шаблонах скрипта (т.е. с большой буквы), в противном же случае
отступы будут расставлены неверно.

Можно использовать этот скрипт и в составе OpenConf. Для этого нужно раскомментировать
первые две строчки, где задаётся имя скриптового движка, и закомментировать строчку
main();
чтобы при старте конфигуратора скрипт не пытался форматировать файлы в каталоге базы 
данных. Для запуска процедуры форматирования нужен вспомогательный скрипт на VBA
RunPerlScripts.vbs.


Запуск скрипта:
\>perl code_beautifier.pl -h
   покажет описание параметров

\>perl code_beautifier.pl -d SRC
   отформатирует все модули в каталоге SRC.

\>perl code_beautifier.pl -f SRC\ГлобальныйМодуль.1S -k
   отформатирует SRC\ГлобальныйМодуль.1S и положит результат в SRC\ГлобальныйМодуль.new.1S.
