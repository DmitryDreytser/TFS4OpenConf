СсылкиНаОбъекты = {"<param1>", "<param2>", "<param3>", "<param2>", {"Refers" [, {"<ID1>"}, {"<ID2>"} ... {"<IDn>"}]}}
	1.	Числовой идентификатор (внутренний)
	2.	Текстовый идентификатор (то, что видно в конфигураторе)
	3.	Комментарий
	4.	Синоним
	ID1 ... IDn - числовые идентификаторы объектов

	Если этот объект входит в состав другого объекта, то первые 4 параметра - пустые. Но они м.б.
	и непустыми, если объект сам по себе просто набор ссылок, как например графа журнала документов.


ПравилаМиграции = {"Distribution ruler", "<param1>", "<param2>", "<param3>"}
Описание правил миграции
	1.	Область распространения: W - все информационные базы, L - место создания, M - место создания и центр
	2.	Коды дополнительных ИБ (поле "дополнительно" в настройках правил миграции): строка
	3.	Автоматическая регистрация изменений: 1 или 0
*примечание: если парметры миграции отсутствуют, то 1С будет считать, что ПравилаМиграции = {"Distribution ruler", "W", "", "1"}

ФормыСписка = {"Forms"} | {"Forms", <Форма> [, <Форма>...]}
Форма = {"<param1>", "<param2>", "<param3>", "<param4>"}

Описание параметров формы
	1.	Числовой идентификатор
	2.	Текстовый идентификатор
	3.	Комментарий
	4.	Синоним

Значения символа, обозначающего тип:
	U:	Неопределенный
	N:	Число
	S:	Строка
	D:	Дата
	E:	Перечисление
	B:	Справочник
	O:	Документ
	C:	Календарь
	A:	ВидРасчета
	T:	Счет
	K:	ВидСубконто
	P:	ПланСчетов

ПравилоПерерасчёта = {"<param1>", ... "<param6>", <ЧтоПересчитывать>, <КогдаПересчитывать>}
ЧтоПересчитывать = СсылкиНаОбъекты
КогдаПересчитывать = СсылкиНаОбъекты

Описание параметров правила перерасчёта
	1.	Числовой идентификатор (внутренний)
	2.	Текстовый идентификатор (то, что видно в конфигураторе)
	3.	Комментарий
	4.	Синоним
	5.	Период для пересчёта: 0 - только в текущем периоде журнала расчётов; 1 - в том же периоде ЖР; 2 - в следующих N периодах
	6.	Когда <param5> = 2, то это число периодов для пересчёта (N)
	7.	Ссылки на виды расчётов, которые нужно пересчитывать
	8.	Ссылки на виды расчётов, при вводе которых происходит пересчёт

	4.	Синоним

Календари

Календари = {"Calendars"} | {"Calendars", <Календарь> [, <Календарь> ...], <КалендарьПраздников>}
Календарь = {"<param1>", ... "<param5>", <НастройкаДней>}
НастройкаДней = {"Days"} | {"Days", "<ДлинаДня1>" [, "<ДлинаДня2>" ...] [, <ПравилаМиграции>]}
	Длины дней задаются в виде целых чисел. Чтобы получить правильную длину, эти числа нужно делить на 100.
КалендарьПраздников = {"HolidaysDef", {"<param1>", ... "<param4>", <ПравилаМиграции>}}

Описание параметров календаря
	1.	Числовой идентификатор (внутренний)
	2.	Текстовый идентификатор (то, что видно в конфигураторе)
	3.	Комментарий
	4.	Синоним
	5.	Стартовая дата в виде числа дней с начала времён
